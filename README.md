
# RAG Chat

> Built a backend microservice to store chat histories generated by a
RAG (Retrieval-Augmented Generation) based chatbot system.

---

## 🚀 Getting Started

### 🛠️ Clone the Repo

To spin up the application locally using Docker:
```bash
git clone https://github.com/Yuvagiri/RagChat_App.git
````

### 🛠️ Local Setup

> For Local setup `.env` file provided gets picked up automatically

```bash
cd RagChat_App
docker compose up
````

---

### 🧪 Dev & Prod Setup

You can start the application in **dev** or **prod** environments using environment-specific `.env` files.

#### Development:

```bash
docker compose --env-file .env.dev up
```

#### Production:

```bash
docker compose --env-file .env.prod up
```

> Make sure `.env.dev` and `.env.prod` files are present in the root directory and contain the correct environment variables.

## 📬 API Endpoints

Use the following endpoints to **create sessions** and **send messages**:

* `GET /actuator/health` - Check the Health Status
* `POST /rag_chat_storage/sessions/create` - Create a new Session
* `DEL /rag_chat_storage/sessions/delete/{session_id}` - Delete the Session and its associated messages
* `PATCH /rag_chat_storage/sessions/105/favorite?isFavorite={true/false}` - Mark / Unmark the Sessions as favourite
* `GET /rag_chat_storage/sessions/getAll?UserName={user_name}` - Get all the Session (get all sessions associated with a user)
* `PATCH /rag_chat_storage/sessions/{session_id}/rename` - Rename a Session
* `POST /rag_chat_storage/messages` - Create a Message (From both the Senders (bot & user)
* `GET /rag_chat_storage/messages/{session_id}` - Get all the Messages for a Session

Details information on requests,headers are provided in [api_documentations](https://github.com/Yuvagiri/rag_chat/blob/dev/API_Documentation.md)

## 📁 Technical Features

```
* Rate Limiting implemented to avoid abuse of API endpoint (5 hits in a min is allowed)
* Centralized Logging implemented as of now in console logs, can be integrated with cloudwatch or other tools later
* Global Error handling implemented for all the API's and operations
* Project Dockerized for local, dev, prod setup with specific .env file. Jar files are being built dynamically for production ready deployments
* Health Check implemented
* PGadmin dockerized for easy access (creds are stored in .env) available at http://localhost:5050/
* Swagger UI Implementation available at http://localhost:8080/swagger-ui/index.html
* CORS implemented, provided option to change or update during prod deployment
* Unit Tests added
```
